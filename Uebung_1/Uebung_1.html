1.1 World Wide Web

  1. Mit welchen fachlichen Argumenten wurde das WWW-Proposal von Tim Burners Lee abgelehnt?

    • Der Chef von Tim Burners Lee soll die Vagheit mit der Evolvierbarkeit verwechselt haben
    • Die Hypertext-Community hatte den Eindruck, dass sie in der Wissenschaft schon weiter fortgeschritten war und dass das Konzept von Tim Berners-Lee ein Rückschritt gewesen wäre. Besonders weil in dem Konzept von Tim Berners-Lee Broken-Links möglich sind (Links die zu nichts führen).

  2. Was sind die fachlichen Argumente, warum das WWW dennoch ein Erfolg wurde?

    • Dezentralisierung
    • Skalierbarkeit
    • Einfache Benutzbarkeit
    • Wirtschaftliche Chancen

  3. Was wäre der Preis für die garantierte Verhinderung von "broken links"?

    • Mit der Gewährleitung der referentiellen Integrität, die technisch gesehen hätte implementiert werden können, bräuchte man eine zentrale Kontrolle (sind Links und Ressourcen auf dem neuesten Stand?)
    • Geschlossenheit


1.2 HTTP

  1. Sie bekommen im Browser den HTTP Status Code 200. Was bedeutet das?
  2. Sie bekommen im Browser den HTTP Status Code 301. Was hat das zu bedeuten?
  3. Sie bekommen im Browser den HTTP Status Code 400. Was hat das zu bedeuten? Was können Sie dagegen tun?
  4. Sie bekommen im Browser den HTTP Status Code 403. Was hat das zu bedeuten? Was können Sie dagegen tun?
  5. In einer Webanwendung benötigen Sie eine OPTIONS-Anfrage, die die Optionen des Servers vor dem eigentlichen Zugriff erfragen soll. Aus Performancegründen soll die Abfrage aber cacheable sein. Wie könnten Sie dafür eine Lösung angehen?

  1. Die Anfrage war erfolgreich und der Server hat zufriedenstellen geantwortet.
  2. Die angeforderte URL wurde dauerhaft an eine andere URL weitergeleitet. Man soll die neuere URL in zukünftigen Anfragen verwenden.
  3. Die Anfrage des Browsers ist fehlerhaft oder ungültig (Möglicherweise fehlende oder ungültige Daten in der Anfrage / ist die vom Browser gesendete Anfrage korrekt und gültig?).
    -> Anfrage überprüfen/schauen ob sie den Anforderungen des Servers entspricht.
  4. Der Server hat die Anfrage verstanden, darf aber aus rechtlichen oder anderen Zugriffs-Gründen nicht auf die Ressource zugreifen (Fehlende Berechtigungen/Zugriffsrechte).
    -> Die fehlenden Berechtigungen/Zugriffsrechte holen.
  5. Man könnte den Cache-Control-Header dazu nutzen, um anzugeben wie lange die OPTIONS-Anfrage gecacht werden soll.

1.6 HTML-Literatur lesen und Fragen beantworten

1.Was ist HTML?

HTML ist eine Auszeichnungssprache. Sie dient dazu Textbasierte Inhalte eines Webdokumentes zu strukturieren.

2.Wie kann man eine geschachtelte geordnete Liste der Tiefe 3 erzeugen?

  1. erster Punkt
  2. zweiter Punkt
    1. erster Unterpunkt von Punkt zwei
    2. zweiter Unterpunkt von Punkt zwei
    3. dritter Unterpunkt von Punkt zwei
      1. Unterpunkt des dritten Unterpunktes
  3. dritter Punkt

3.Wie ist eine HTML-Tabelle aufgebaut?

Man nutzt den Tag <table>. In diesem gibt man mit <th> den Spaltenname an und mit <td> (=table data) den Inhalt der Tabellenzelle. Jede Tabellenreihe fängt mit <tr> (=table row) an und hört mit </tr> auf.

4.Welche Konventionen sollte man bei Pfaden und Dateinamen beachten?

Die Dateien müssen von dem Server akzeptiert werden.

  1. Deutsche Umlaute vermeiden
  2. Scharfes S vermeiden
  3. Dateiname nicht länger als 256 Zeichen
  4. ?,* und = dürfen auf keinen Fall benutzt werden.
  5. Auf das Leerzeichen verzichten!
  6. Unter den Sonderzeichen ist auf jeden fall das "_" erlaubt.

5.Wie baut man in HTML ein Menü?

  • Startseite
  • Dienstleistungen
  • Impressum

6.Welche Attribute sollte man bei Bildern wie verwenden?

"src" um zu sagen wo das Bild überhaupt ist, mit "alt" evtl. eine Beschreibung des Bildes, "title" für den Titel des Bildes und evtl. noch "width" und "height" für die Breite und Höhe des Bildes. Man kann auch einfach nur eines der beiden angeben. Dann wird das andere Attribut entsprechend skaliert, sodass das Verhältnis noch stimmt.
